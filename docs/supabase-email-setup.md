# Supabase 이메일 설정 가이드

## 개발 환경에서 테스트 이메일 사용하기

Supabase에서 `test@test.com` 같은 테스트 이메일을 사용하려면 Supabase 대시보드에서 이메일 확인을 비활성화해야 합니다.

## 설정 방법

### 1. Supabase 대시보드 접속
1. [Supabase Dashboard](https://supabase.com/dashboard)에 로그인
2. 프로젝트 선택

### 2. 이메일 확인 비활성화 (개발 환경용)
1. **Authentication** → **Providers** 메뉴로 이동
2. **Email** 제공업체 선택
3. **Email Confirmation** 설정 찾기
4. **Enable email confirmations** 체크 해제
   - 개발 환경에서는 비활성화
   - 프로덕션에서는 반드시 활성화해야 합니다

### 3. 이메일 검증 설정 확인
1. **Authentication** → **Settings** 메뉴로 이동
2. **Email** 설정 확인
3. **Email validation** 설정 확인
   - 기본적으로 이메일 형식만 검증합니다
   - 실제 도메인 존재 여부는 검증하지 않습니다

## 주의사항

### 개발 환경
- 이메일 확인을 비활성화하면 `test@test.com` 같은 테스트 이메일도 사용 가능합니다
- 개발 환경에서만 사용하고, 프로덕션에서는 반드시 활성화해야 합니다

### 프로덕션 환경
- 이메일 확인을 활성화하여 보안을 강화해야 합니다
- 실제 이메일 주소만 사용해야 합니다
- 테스트 이메일은 사용하지 않아야 합니다

## 대안: 실제 이메일 사용

개발 환경에서도 실제 이메일을 사용하는 것을 권장합니다:
- Gmail, 네이버 등 실제 이메일 서비스 사용
- 임시 이메일 서비스 사용 (10minutemail.com 등)

## 문제 해결

### `email_address_invalid` 에러가 발생하는 경우
1. Supabase 대시보드에서 이메일 확인 설정 확인
2. 이메일 형식이 올바른지 확인
3. 실제 이메일 주소를 사용해보기
4. Supabase 로그 확인 (Dashboard → Logs → Auth)

### 이메일 확인이 필요한 경우
1. 이메일 확인을 활성화한 경우, 회원가입 후 이메일을 확인해야 로그인 가능
2. 개발 환경에서는 이메일 확인을 비활성화하는 것을 권장
